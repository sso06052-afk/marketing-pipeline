# 음원 홍보 파이프라인

> 멜론·지니 신곡 페이지에서 신규 가수를 자동 수집하고,  
> 인스타그램 계정을 찾아 DM 발송 및 답장 관리까지 한 곳에서 처리하는 시스템입니다.

---

## 무엇이 자동으로 처리되나요?

| 작업 | 기존 방식 | 이 시스템 |
|---|---|---|
| 신곡 가수 발굴 | 멜론·지니 페이지를 매일 직접 확인 | 버튼 한 번으로 자동 수집 |
| 인스타 계정 찾기 | 가수마다 직접 구글링 (5~10분/명) | 자동 탐색 + AI 판별 (수 초/명) |
| 발송 관리 | 엑셀이나 메모장으로 수기 관리 | DM 클릭 → 자동 기록 |
| 답장·계약 추적 | 별도 도구 사용 | 대시보드에서 통합 관리 |
| 통계 | 직접 집계 | 자동 집계 (확보율·발송률·답장률·계약률) |

---

## 시스템 요구사항

| 항목 | 최소 | 권장 |
|---|---|---|
| OS | Mac (10.15+) / Windows 10, 11 | 최신 버전 |
| 메모리 | 8GB | 16GB |

## 설치

처음 사용하시는 분은 아래 두 가이드를 순서대로 따라하세요.

### 1️⃣ **[API 키 발급 가이드 →](docs/api-keys-setup.md)**
Supabase, Gemini, Serper 3가지 서비스 가입 + 키 발급 (약 30~40분)

### 2️⃣ **[설치 및 사용 가이드 →](docs/client-setup-guide.md)**
Docker 설치 → 환경변수 입력 → 대시보드 실행 (약 30~60분)

전체 흐름 요약:
1. **API 키 발급** (Supabase, Gemini, Serper) — 무료
2. **Docker Desktop** 설치 (무료, 최초 1회)
3. **`.env` 파일** 생성 (발급받은 키 입력)
4. 터미널에서 명령어 2줄 실행
5. 브라우저에서 `localhost:3000` 접속

총 소요 시간: **약 1~2시간**

## 빠른 명령어 참고

설치가 완료된 이후 일상적으로 쓰는 명령어입니다.

```bash
# 대시보드 켜기 (컴퓨터 재시작 후)
docker start pipeline-dashboard

# 대시보드 끄기
docker stop pipeline-dashboard

# 로그 확인 (문제 생겼을 때)
docker logs pipeline-dashboard

# 상태 확인
docker ps
```

브라우저 접속: `http://localhost:3000`

---

## 매일 사용 흐름

1. Docker Desktop 실행 (작업표시줄 고래 아이콘 확인)
2. 브라우저로 `localhost:3000` 접속
3. 우측 상단 **▶ 실행** 버튼 → 신곡 수집 (1~2분)
4. **발송대기** 탭에서 가수 확인 → **DM** 버튼 클릭
5. 답장 오면 가수 행 클릭 → **답장완료** + 결과(긍정/거절/보류) 입력

## 자주 묻는 질문

**Q. 컴퓨터를 끄면 어떻게 되나요?**  
→ 대시보드도 꺼집니다. 다시 켜면 자동 재시작되도록 설정되어 있습니다.

**Q. 다른 사람과 데이터 공유가 되나요?**  
→ 데이터는 클라우드에 저장됩니다. 다른 컴퓨터에 설치해도 같은 데이터가 보입니다.  
다만 **동시에 두 곳에서 수집을 돌리지 마세요** (중복 발생 가능).

**Q. 매일 컴퓨터를 켜둬야 하나요?**  
→ 사용할 때만 켜시면 됩니다. 자동 수집까지 원하시면 별도 서버 배포가 필요합니다(담당자 문의).

**Q. 데이터가 안 보여요**  
→ 1) Docker Desktop이 실행 중인지 / 2) `docker ps`로 컨테이너가 떠있는지 확인  
   해결 안 되면 [가이드 — 자주 묻는 문제](docs/client-setup-guide.md#자주-묻는-문제) 섹션 참조
