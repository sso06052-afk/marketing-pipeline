# UX 리디자인 플랜 — 2페이지 컴팩트 구조

> 작성일: 2026-04-29  
> 목표: 4페이지 → 2페이지, 중복 제거 + UX 품질 최고로

---

## 현황 진단

| 페이지 | 문제 |
|---|---|
| `/` 발송 리스트 | 파이프라인 실행 + 탭 4개 (발송대기/발송완료/답장완료/검토필요) |
| `/review` 인스타 미확인 | "검토필요" 탭과 **100% 중복** — 카드 UI만 다름 |
| `/db` 전체 DB | 발송 리스트와 같은 데이터, 다른 필터 UI |
| `/stats` 통계 | 유일하게 고유하나 독립 페이지까지 필요 없음 |

**핵심 중복:** `/review` ⊆ 검토필요 탭, `/stats` → `/db`에 통합 가능

## 최종 구조: 2페이지

```
/ (대시보드)       — 오늘 할 일 + 파이프라인 실행
/db (전체 관리)    — 전체 검색·필터·통계
```

### Nav 변경
```
전: 홍보 파이프라인 | 발송 리스트 | 인스타 미확인 | 전체 DB | 통계
후: 홍보 파이프라인 | 대시보드 | 전체 관리
```

## 페이지 1: `/` 대시보드

### 레이아웃
```
┌─────────────────────────────────────────┐
│ 오늘 날짜        [멜론|지니] [▶ 실행]   │  ← 헤더 (현재와 동일)
├─────────────────────────────────────────┤
│ [파이프라인 로그 패널 — 실행 시만 표시] │
├──────┬──────┬──────┬──────┐             │
│ 신규 │인스타│발송  │답장  │             │  ← 오늘 통계 카드 4개
└──────┴──────┴──────┴──────┘
│ 발송대기(N) │ 검토필요(🔴N) │ 발송완료 │ 답장완료 │  ← 탭
├─────────────────────────────────────────┤
│  아티스트 테이블 or 카드 (탭에 따라)    │
└─────────────────────────────────────────┘
```

### 탭 UX 개선

**발송대기 탭** (기본)
- 현재 ArtistRow 테이블 유지
- DM 버튼 → 원클릭 (클립보드 복사 + 인스타 열기 + DB 업데이트 동시)

**검토필요 탭** (🔴 배지)
- `/review` 페이지의 카드 UI 흡수 — 테이블보다 카드가 훨씬 편함
- 카드당: 가수명 + 곡명 + 멜론링크 + @handle 입력 + 저장
- 저장 후 즉시 카드 사라짐 (낙관적 업데이트)
- 배지: `tabCounts.review > 0` 이면 빨간 dot 표시

**발송완료 탭**
- 현재 그대로 (ArtistRow, 답장 버튼 포함)

**답장완료 탭**
- reply_result(긍정/보류/거절) 뱃지 컬러로 구분
- 메모 inline 표시

### 삭제
- `/review` 페이지 전체 삭제 (기능은 검토필요 탭으로 흡수)

## 페이지 2: `/db` 전체 관리

### 레이아웃
```
┌─────────────────────────────────────────┐
│ 전체 관리           [목록] [통계]        │  ← 상단에 2개 탭
├─────────────────────────────────────────┤
│ [검색] [상태 칩] [정렬] [초기화]  총 N명 │  ← 필터 바
├─────────────────────────────────────────┤
│ 아티스트 테이블 (현재 /db와 동일)        │
└─────────────────────────────────────────┘
```

### UX 개선

**검색 — debounce 적용**
- 현재: 타이핑 후 Enter 또는 버튼 클릭 필요
- 변경: 300ms debounce → 타이핑하면 바로 필터됨

**상태 필터 — select → 칩(chip) 버튼**
- 현재: `<select>` 드롭다운
- 변경: 클릭형 칩 버튼 (전체 / 발송대기 / 발송완료 / 답장완료 / 검토필요)
- 시각적으로 현재 선택 상태가 명확, 1클릭으로 전환

**통계 탭 통합**
- `/stats` 페이지 삭제
- `/db`의 "통계" 탭으로 흡수 (현재 stats/page.tsx 내용 그대로)

### 삭제
- `/stats` 페이지 전체 삭제 (기능은 /db 통계 탭으로 흡수)
